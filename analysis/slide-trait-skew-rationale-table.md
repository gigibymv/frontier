# Slide Trait Skew Rationale Table

Purpose: provide a slide-ready classification table for the exact Infrastructure, Platform, and Operating Capabilities traits. Each row says whether the trait is skewed, why it is or is not startup-skewed, and why it is or is not large-org-skewed.

Classification labels:

- `Startup-skewed`: stronger signal in startup / scale-up operating patterns, but not exclusive.
- `Large-org-skewed`: stronger signal in enterprise / large-organization operating patterns, but not exclusive.
- `Both / not specific`: meaningful on both sides; do not mark as startup- or large-org-specific.
- `Preliminary`: plausible or supported by a strong example, but low-n or too narrow for a strong slide mark.
- `Do not promote`: insufficient transcript support, likely deck-only / external research, or previously flagged as unsupported.

## Use Guidance

For slide marking, the safest highlighted traits are:

- **Startup-skewed:** `Context lives close to work`, `Explicit task plan / spec file`, `High-reasoning model for planning`, `Acceptance criteria / edge cases`, `Core engineering skill shifting from coding to orchestration`, `Committer owns AI output`, `Model routing by task complexity and cost`, `Fixes -> skills`.
- **Large-org-skewed:** `Just-in-time context via governed store`, `Agent identity & access managed centrally`, `Pre-wired sequences (deterministic)`, `Ownership, execution, & signoffs per workflow`, `Audit trails, ownership, & revocation`, `Formal context ownership responsibility`, `Limit token burn until definitive value realization`.

Keep `Both / not specific`, `Preliminary`, and `Do not promote` traits visually distinct from the strongest skewed claims.
