# Context Ownership

## Definition

Context ownership is the idea that someone should explicitly own the freshness, scope, and approval boundary of the context that AI systems consume.

## Evidence Summary

- The current seed evidence is one direct enterprise transcript reference that describes context ownership as an extension of metric ownership.
- The pattern is promising but not yet broadly evidenced in this corpus.

## Open Questions

- Is context ownership a stable role, a rotating responsibility, or a temporary transition mechanism?
- What are the review boundaries for updating shared context?
- How should context ownership be separated from data ownership, workflow ownership, and platform ownership?
