# Platform Context Layer

## Definition

The platform context layer is the set of mechanisms that assemble, shape, permission, and refresh the context that workflows or agents use at runtime.

## Evidence Summary

- This concept clearly matters in the corpus, but the current quote-span-backed material is still mostly adjacent Operating Capabilities evidence.
- Current linked claims cover context ownership and engineering/permissioned platform rails; neither should yet be treated as a complete Platform maturity model.

## Open Questions

- Where is the line between shared platform context and domain-owned local context?
- Which parts of context assembly belong to Platform versus Infrastructure?
- How should context freshness, permissions, and provenance be represented in the benchmark?
