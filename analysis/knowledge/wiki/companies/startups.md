# Startups

## Current Synthesis

Startup-scaleup and startup-folder evidence currently supplies many of the strongest examples for role-level tool building, agent orchestration, customer-facing human gates, and workflow-specific role redesign. Token controls should be handled as mixed organization-type evidence because one anchored source is enterprise-classified in the registry.

## Evidence-Backed Patterns

- AI fluency and orchestration as role expectations: `clm-oc-ai-fluency-as-threshold`
- Workflow-owner-led automation: `clm-oc-workflow-owners-build-local-automations`
- Accountability-based human control: `clm-oc-accountability-based-human-control`
- Experimentation governance: `clm-oc-experimentation-budget-governance`
- Mixed organization-type token controls: `clm-oc-token-budgeting-by-use-case`

## Caveats

- Startup evidence can show advanced implementation style, but not all patterns are startup-only.
- The transcript folder is not always the same as registry `company_type`; use `../../sources/source_registry.yaml` before making startup-only claims.
- Avoid exclusive labels unless the corpus proves exclusivity.
